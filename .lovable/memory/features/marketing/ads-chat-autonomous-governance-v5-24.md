# Memory: features/marketing/ads-chat-autonomous-governance-v5-24
Updated: 2026-03-11

O Ads Chat (v5.33.0) utiliza uma arquitetura de dados em tempo real e execução rigorosa. 1. Fonte de Dados: Prioriza a consulta direta via Meta Graph API (fetchMetaCampaignsLive) com paginação total. O default de `getCampaignPerformance` agora é `date_preset=maximum` (LIFETIME). O banco local é usado apenas como fallback. 2. Conversões — MAX Value Deduplication: O parser itera todos os 8 action_types de purchase e usa o que tiver o MAIOR valor (`if (val > conversions) conversions = val`). NÃO soma tipos diferentes (causa inflação) e NÃO usa prioridade fixa (causa subestimação em contas onde omni_purchase é menor que purchase). 3. Pipeline Anti-Promessa DUAL: A IA é proibida de encerrar turnos com frases de espera sem disparar ferramentas. Existem DOIS pontos de detecção de filler: (a) na resposta INICIAL (sem tool calls) e (b) DENTRO do tool loop (após executar uma ferramenta, a IA responde com filler em vez de chamar mais ferramentas). Ambos detectam 15+ padrões de filler (incluindo "estou realizando", "esse processo envolve", "múltiplas chamadas à API", "assim que tiver os dados") e forçam retry com tool_choice=required. A v5.33.0 corrigiu o bug onde filler dentro do tool loop não era detectado. 4. Feedback: Eventos SSE de progresso. 5. Integridade: Nomes exatos, proibindo abreviações. 6. Imagens: Ctrl+V para colar screenshots. 7. Targeting — Sync & Cache: `fetchMetaAdsetsLive` faz paginação COMPLETA (até 500 adsets) sem timeout artificial, com upsert fire-and-forget de cada adset (targeting JSONB) no `meta_ad_adsets`. `getAdsetTargeting` busca até 20 adsets sem timeout, com retry automático em rate limit (429). Cada resultado é cacheado localmente via `syncAdsetToCache()`. 8. Fluxo de Targeting: O system prompt instrui AÇÃO IMEDIATA — proíbe descrever etapas sem chamar ferramentas. Passo 1: `get_meta_adsets` (sem live). Passo 2: `get_adset_targeting` com IDs específicos.
