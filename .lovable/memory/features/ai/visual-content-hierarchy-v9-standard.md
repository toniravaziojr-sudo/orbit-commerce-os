# Memory: features/ai/visual-content-hierarchy-v9-standard
Updated: now

A hierarquia de geração de imagens (v9.1) prioriza a fidelidade visual absoluta para produtos e anúncios. 1. Provedor Primário: 'fal-ai/gpt-image-1/edit-image' (GPT Image 1) é o modelo mandatório para variações de produto, pois utiliza a imagem de referência para manter cores, rótulos e formatos idênticos ao original. Timeout reduzido para 60s (era 120s) para acelerar fallback. 2. Fallbacks: Gemini Nativa (via base64), seguido por OpenAI Nativa e Lovable AI Gateway — TODOS recebem imagem real do produto + prompt criativo completo. 3. Pipeline Assíncrono: O processo segue o padrão Submit -> Poll -> Link. Edge functions (ex: 'creative-image-generate') retornam '202 Accepted' com um 'job_id' e o frontend realiza polling na tabela 'creative_jobs' (ex: a cada 4s) para monitorar o status. 4. Restrição: O modelo FLUX 2 (Pro/Turbo) foi removido dos fluxos de produtos por ser estritamente text-to-image e gerar resultados genéricos sem suporte real a imagem de referência. 5. Regra inviolável: o produto com fidelidade visual é inegociável em todos os níveis de fallback.
